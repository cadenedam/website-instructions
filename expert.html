Project vs User site

Project pages can be published from /docs or a dedicated branch such as gh-pages — choose based on your workflow. If you want a single central site for a class, a user site (`USERNAME.github.io`) is the simplest. :contentReference[oaicite:10]{index=10}

React apps (short)

If you build with React, the common pattern is to add a homepage in package.json and use the gh-pages package to publish the built build/ folder. (Example below.) :contentReference[oaicite:11]{index=11}

// package.json snippet
"homepage": "https://USERNAME.github.io/REPO-NAME",
"scripts": {
  "start": "react-scripts start",
  "build": "react-scripts build",
  "predeploy": "npm run build",
  "deploy": "gh-pages -d build"
}

Custom domain

Add a CNAME file or set the domain in Pages settings; GitHub can create the CNAME commit for you. Remember to update DNS records at your domain registrar. :contentReference[oaicite:12]{index=12}

Enforce HTTPS

Turn on "Enforce HTTPS" in Pages settings after your site is published. :contentReference[oaicite:13]{index=13}
